# Phase 2.5 — Overlay-Pixel Fine Mask Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development or superpowers:executing-plans. Continues on `feat/phase2-water`.

**Goal:** Pixel-accurate water detection — the mask follows the painted water silhouette (the overlay tile's non-transparent pixels), replacing cell-resolution flags + color heuristics.

**Design (locked, decompile- and user-validated):** Flagged WED cells alpha-blend `wt*`/`ys*` overlay tiles; the overlay tile's non-transparent pixels ARE the water contour. Build: parse the WED overlay tilemaps (cell -> overlay tile index), decode each unique overlay tile's transparency (classic palette TIS), stamp a sub-cell fine mask (8px/texel, texel = TileLiquidMode or 0), upload via the existing unit-2 path (worldSizeInv math is resolution-independent), and delete the shader's coverage bilinear + luma/chroma gates.

**Tasks:**

1. **WED overlay tilemaps (host, TDD)** — extend `wed_runtime` to expose, per liquid overlay: dimensions + per-cell overlay tile index (tilemap entry -> startingTile -> tile-index lookup). Synthetic-WED tests beside the existing `parse_loaded_wed` tests.
2. **Palette tile transparency (host, TDD)** — `decode_palette_tile_alpha(bytes) -> 64x64 opacity bits`. Classic palette tile = 1024B BGRA palette + 4096 index bytes. Transparent = palette color is the green key (0,255,0) or palette index 0. Tests with synthetic tiles.
3. **Fine-mask stamper (host, TDD)** — `build_fine_liquid_mask(wed, overlayTiles, tileAlphaProvider) -> AreaCellTexture` at 8px/texel (w*8 x h*8): texel = overlay's TileLiquidMode where the 8x8 pixel block is majority-opaque, else 0. Cells without decodable overlay tiles fall back to full-cell mode (current behavior).
4. **Runtime tile fetch + upload (Windows)** — read overlay tile bytes from `CInfinity.pTileSets[N]` (CInfTileSet.pResTiles[i] -> CRes.pData, all safe_read; offsets verified against EEex docs/decompile), decode/stamp at `refresh_wed_cache`, upload the fine mask on unit 2 (NEAREST+CLAMP). Cell-resolution mask remains the fallback on any failure.
5. **Shader simplification** — sample the fine mask directly (center tap = mode; 4 neighbor taps at 8px for soft edge alpha + shore factor); DELETE `ieeCoverage` bilinear, the luma gate, and the chroma gate.
6. **Runbook + CI** — v15 session: arrow-test the contour, fountains, map changes, a lava/sewage area if reachable.

**Out of scope:** base-TIS-painted water with no overlay (separate problem, needs page decode); visual polish tuning (after edges are clean).
